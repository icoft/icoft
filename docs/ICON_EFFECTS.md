# Icon Effects Specification

This document outlines the visual effects that can be applied to icons generated by Icoft. These effects are designed to enhance icon visibility across different platforms and themes.

## Overview

Icoft generates icons with transparent backgrounds by default. While this works well for many use cases, it can cause visibility issues when:
- The icon foreground color matches the system background (e.g., white icon on light theme)
- The icon is displayed on varying backgrounds across different platforms

To address these issues, Icoft provides a range of visual effects that can be applied during icon generation.

## Effect Categories

### 1. Background Effects

Effects that modify the icon background.

#### 1.1 Solid Color Background
- **Description**: Fills the entire icon canvas with a solid color
- **Use Case**: Ensures icon visibility on any background
- **Parameters**:
  - `color`: Hex color code (e.g., `#4A90D9`)
  - `padding`: Space between icon content and background edge
- **CLI Example**: `--bg-color "#4A90D9"`

#### 1.2 Gradient Background
- **Description**: Fills background with a gradient
- **Types**:
  - Linear (top-to-bottom, left-to-right, diagonal)
  - Radial (center-outward)
  - Angular (sweep gradient)
- **Parameters**:
  - `type`: `linear`, `radial`, `angular`
  - `colors`: Array of color stops (e.g., `["#4A90D9", "#7B68EE"]`)
  - `angle`: Rotation angle for linear gradients
- **CLI Example**: `--bg-gradient "linear:#4A90D9:#7B68EE:45"`

#### 1.3 Custom Background Image
- **Description**: Uses user-provided image as background
- **Parameters**:
  - `image_path`: Path to background image
  - `fit`: `cover`, `contain`, `stretch`
  - `opacity`: Background image opacity (0-1)
- **CLI Example**: `--bg-image "bg.png" --bg-fit cover`

### 2. Border Effects

Effects that add borders around the icon content.

#### 2.1 Solid Border
- **Description**: Draws a solid-color border along the icon's alpha edge
- **Parameters**:
  - `color`: Border color
  - `width`: Border thickness in pixels
  - `style`: `solid`, `dashed`, `dotted`
- **CLI Example**: `--border-color "#333" --border-width 3`

#### 2.2 Gradient Border
- **Description**: Border with gradient color
- **Parameters**:
  - `colors`: Gradient color stops
  - `width`: Border thickness
  - `type`: `linear`, `radial`
- **CLI Example**: `--border-gradient "linear:#FF0000:#00FF00"`

#### 2.3 Double Border
- **Description**: Two concentric borders for maximum visibility
- **Parameters**:
  - `inner_color`: Inner border color
  - `outer_color`: Outer border color
  - `width`: Total border width
- **Use Case**: White icon gets dark outer + light inner border
- **CLI Example**: `--border-double "#FFF:#333"`

### 3. Shadow Effects

Effects that add depth through shadows.

#### 3.1 Drop Shadow
- **Description**: Shadow offset from the icon
- **Parameters**:
  - `color`: Shadow color (default: `#00000080`)
  - `offset_x`: Horizontal offset
  - `offset_y`: Vertical offset
  - `blur_radius`: Shadow blur amount
  - `spread`: Shadow size expansion
- **CLI Example**: `--shadow "#00000040:5:5:10"`

#### 3.2 Inner Shadow
- **Description**: Shadow inside the icon shape (inset effect)
- **Parameters**:
  - `color`: Shadow color
  - `blur_radius`: Blur amount
  - `offset_x`, `offset_y`: Offset direction
- **CLI Example**: `--shadow-inner "#00000060:3:0:0"`

#### 3.3 Glow Effect
- **Description**: Outer glow around the icon
- **Parameters**:
  - `color`: Glow color
  - `radius`: Glow spread radius
  - `intensity`: Glow brightness
- **CLI Example**: `--glow "#4A90D9:20:0.8"`

### 4. Shape Masks

Effects that constrain the icon to specific shapes.

#### 4.1 Rounded Rectangle
- **Description**: macOS-style rounded rectangle background
- **Parameters**:
  - `radius`: Corner radius
  - `color`: Background color
- **CLI Example**: `--shape-rounded "20:#4A90D9"`

#### 4.2 Circle
- **Description**: Circular background/container
- **Parameters**:
  - `color`: Background color
  - `padding`: Space between icon and circle edge
- **CLI Example**: `--shape-circle "#4A90D9:10"`

#### 4.3 Squircle
- **Description**: Superellipse shape (modern iOS/macOS style)
- **Parameters**:
  - `curvature`: Shape curvature factor
  - `color`: Background color
- **CLI Example**: `--shape-squircle "0.5:#4A90D9"`

### 5. Advanced Effects

#### 5.1 Glassmorphism
- **Description**: Frosted glass effect with blur
- **Parameters**:
  - `blur`: Background blur amount
  - `opacity`: Background opacity
  - `tint`: Color tint
- **CLI Example**: `--effect-glass "10:0.7:#FFFFFF"`

#### 5.2 Neon Glow
- **Description**: Bright neon-style glow
- **Parameters**:
  - `color`: Neon color
  - `intensity`: Glow strength
  - `pulse`: Animated pulse (for SVG only)
- **CLI Example**: `--effect-neon "#FF00FF:1.5"`

#### 5.3 3D Emboss
- **Description**: Raised or recessed 3D effect
- **Parameters**:
  - `type`: `raised`, `recessed`
  - `highlight`: Highlight color
  - `shadow`: Shadow color
  - `depth`: 3D depth intensity
- **CLI Example**: `--effect-3d "raised:#FFF:#000:3"`

## Implementation Phases

### Phase 1: Core Background Effects (P0)
- Solid color background
- Linear gradient background
- Basic padding/margin control

**Estimated Effort**: 2-3 days
**Dependencies**: Pillow ImageDraw, ImageFilter

### Phase 2: Borders and Shadows (P1)
- Solid border (alpha edge detection)
- Drop shadow
- Inner shadow
- Double border for visibility

**Estimated Effort**: 3-4 days
**Dependencies**: ImageFilter.MaxFilter, Gaussian blur

### Phase 3: Shape Masks (P1)
- Rounded rectangle
- Circle mask
- Squircle (superellipse)

**Estimated Effort**: 2-3 days
**Dependencies**: ImageDraw rounded_rectangle, mask operations

### Phase 4: Advanced Effects (P2)
- Glassmorphism (blur + opacity)
- Glow effects
- 3D emboss
- Custom background images

**Estimated Effort**: 4-5 days
**Dependencies**: Advanced ImageFilter, potential SVG filters

## Technical Considerations

### Performance
- Effects should be applied at the final rendering stage
- Cache intermediate results when possible
- Consider effect complexity for large icon sets (10+ sizes)

### Platform Compatibility
- **Windows**: ICO format supports all effects
- **macOS**: ICNS format supports all effects; system applies rounded corners
- **Linux**: PNG format supports all effects
- **Web**: PNG/SVG support all effects; SVG allows animated effects

### Color Space
- All effects should support RGBA color space
- Consider color blindness accessibility for auto-generated borders

## CLI Design

### Option Grouping
```bash
# Background options
--bg-color COLOR
--bg-gradient TYPE:COLOR1:COLOR2[:ANGLE]
--bg-image PATH [--bg-fit MODE] [--bg-opacity VALUE]

# Border options  
--border-color COLOR [--border-width PIXELS]
--border-gradient TYPE:COLOR1:COLOR2
--border-double INNER:OUTER

# Shadow options
--shadow COLOR:OFFSET_X:OFFSET_Y:BLUR
--shadow-inner COLOR:BLUR:OFFSET_X:OFFSET_Y
--glow COLOR:RADIUS:INTENSITY

# Shape options
--shape-rounded RADIUS[:COLOR]
--shape-circle [COLOR] [--shape-padding PIXELS]
--shape-squircle CURVATURE[:COLOR]

# Preset combinations
--icon-style STYLE  # e.g., "ios", "material", "glass"
```

### Preset Styles
- `ios`: Rounded rectangle with subtle shadow
- `material`: Circle with elevation shadow
- `glass`: Glassmorphism with blur
- `minimal`: Thin border, no background
- `adaptive`: Auto-detects icon color, adds contrasting border

## Examples

### Example 1: White Icon on Blue Background
```bash
icoft -a --bg-color "#4A90D9" white_logo.png icons/
```

### Example 2: Gradient Background with Shadow
```bash
icoft -a --bg-gradient "linear:#4A90D9:#7B68EE:45" \
       --shadow "#00000040:4:4:8" \
       logo.png icons/
```

### Example 3: macOS-Style Rounded Icon
```bash
icoft -a --shape-rounded "22%:#FFFFFF" \
       --shadow "#00000020:0:4:12" \
       logo.png icons/
```

### Example 4: Adaptive Visibility Border
```bash
icoft -a --border-double "auto" logo.png icons/
# Auto-detects icon color, adds contrasting double border
```
